# Как редактировать документацию

Любой пользователь может помочь улучшить документацию ArtGeneration.me! Это проще, чем кажется. Здесь вы найдете пошаговую инструкцию для новичков.

## Зачем это нужно?

- Вы заметили ошибку или опечатку
- Описание устарело после обновления сервиса
- Хотите добавить полезную информацию
- Можете лучше объяснить какую-то функцию

Ваш вклад поможет тысячам других пользователей!

## Способ 1: Редактирование через браузер (самый простой)

Этот способ не требует установки программ. Всё делается в браузере.

### Шаг 1: Найдите страницу для редактирования

1. Откройте страницу документации, которую хотите исправить
2. Прокрутите вниз страницы
3. Нажмите ссылку **"Редактировать на GitHub"**

### Шаг 2: Войдите в GitHub

Если у вас нет аккаунта GitHub:
1. Перейдите на [github.com](https://github.com)
2. Нажмите **Sign up**
3. Введите email, придумайте пароль и имя пользователя
4. Подтвердите email

Если аккаунт есть — просто войдите.

### Шаг 3: Отредактируйте файл

1. GitHub покажет текст страницы в редакторе
2. Найдите место, которое хотите изменить
3. Внесите правки прямо в текст

### Шаг 4: Сохраните изменения

1. Прокрутите вниз до раздела **"Commit changes"**
2. В поле **"Commit message"** кратко опишите что изменили
   - Например: "Исправил опечатку в разделе генератора"
3. Нажмите зеленую кнопку **"Propose changes"**

### Шаг 5: Создайте Pull Request

1. На следующей странице нажмите **"Create pull request"**
2. Проверьте описание и нажмите **"Create pull request"** ещё раз
3. Готово! Ваши изменения отправлены на проверку

После проверки модератором изменения появятся на сайте.

## Способ 2: Локальное редактирование (для продвинутых)

Этот способ даёт больше контроля и удобен для больших изменений.

### Требования

- Установленный [Git](https://git-scm.com/downloads)
- Установленный [Node.js](https://nodejs.org/) (версия 18 или выше)
- Текстовый редактор ([VS Code](https://code.visualstudio.com/) рекомендуется)

### Шаг 1: Склонируйте репозиторий

```bash
git clone https://github.com/timoncool/artgen_docs.git
cd artgen_docs
```

### Шаг 2: Установите зависимости

```bash
npm install
```

### Шаг 3: Запустите локальный сервер

```bash
npm run dev
```

Откройте в браузере: http://localhost:3000

### Шаг 4: Редактируйте файлы

Файлы документации находятся в папке `pages/`:
- `pages/ru/` — русская версия
- `pages/en/` — английская версия

Изменения автоматически отображаются в браузере при сохранении файла.

### Шаг 5: Создайте Pull Request

```bash
git checkout -b my-changes
git add .
git commit -m "Описание изменений"
git push origin my-changes
```

Затем откройте GitHub и создайте Pull Request.

## Формат файлов (MDX)

Документация написана в формате MDX — это Markdown с поддержкой React-компонентов.

### Основные элементы

```mdx
# Заголовок первого уровня

## Заголовок второго уровня

### Заголовок третьего уровня

Обычный текст параграфа. Можно использовать **жирный текст**
и *курсив*.

- Элемент списка
- Ещё один элемент
- И ещё один

1. Нумерованный список
2. Второй пункт
3. Третий пункт

[Текст ссылки](https://example.com)

![Описание изображения](/screens/example.png)
```

### Выделение важной информации

```mdx
> Это цитата или важное замечание

**Важно:** Это выделенный текст.
```

### Код

Для вставки кода используйте обратные кавычки:

```mdx
Инлайн код: `npm install`

Блок кода:
\`\`\`bash
npm install
npm run dev
\`\`\`
```

## Добавление изображений

### Шаг 1: Подготовьте изображение

- Формат: PNG или WebP
- Рекомендуемая ширина: 800-1200 пикселей
- Сожмите изображение для быстрой загрузки

### Шаг 2: Загрузите в папку

Поместите изображение в папку `public/screens/`

### Шаг 3: Добавьте в документацию

```mdx
![Описание изображения](/screens/имя-файла.png)
```

## Структура документации

```
pages/
├── ru/                    # Русская версия
│   ├── docs/              # Руководство пользователя
│   │   ├── generator/     # Генератор изображений
│   │   ├── gallery/       # Галерея
│   │   ├── editor/        # Редактор
│   │   └── ...
│   ├── news.mdx           # Новости
│   ├── contributors.mdx   # Авторы
│   └── contributing.mdx   # Эта страница
└── en/                    # English version
```

## Советы для авторов

### Пишите понятно

- Используйте простые слова
- Избегайте длинных предложений
- Разбивайте текст на короткие абзацы

### Будьте точны

- Проверяйте факты перед публикацией
- Указывайте конкретные шаги
- Добавляйте скриншоты где уместно

### Соблюдайте стиль

- Обращайтесь к читателю на "вы"
- Используйте активный залог
- Следуйте структуре существующих страниц

## Часто задаваемые вопросы

### Сколько времени занимает проверка?

Обычно от нескольких часов до пары дней.

### Что если мои изменения отклонят?

Модератор объяснит причину и подскажет, как исправить.

### Могу ли я предложить новый раздел?

Да! Создайте Issue на GitHub с описанием идеи.

### Получу ли я что-то за помощь?

Вы можете добавить себя на [страницу авторов](/ru/contributors) и получить благодарность сообщества!
